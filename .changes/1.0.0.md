## 1.0.0 - 2026-04-07


### Breaking

#### Refined the public error API for DETS operations.

**`ErlangError` renamed to `UnexpectedError`** — The `ErlangError(detail)`
variant on `DetsError` is now `UnexpectedError(detail)`, making it explicit
that the payload is diagnostic-only. Replace any `ErlangError` pattern match
arm with `UnexpectedError`:

```gleam
// Before
case err {
  slate.NotFound -> ...
  slate.ErlangError(detail) -> io.println("unexpected: " <> detail)
  _ -> ...
}

// After
case err {
  slate.NotFound -> ...
  slate.UnexpectedError(detail) -> io.println("unexpected: " <> detail)
  _ -> ...
}
```

For stable error classification without matching on the string payload, use
the new `slate.error_code` and `slate.error_message` helpers.

**`update_counter` now returns `UpdateCounterError`** —
`slate/set.update_counter` returns `Result(Int, UpdateCounterError)` instead
of `Result(Int, DetsError)`. The new error type has two variants:
`CounterValueNotInteger` (the stored value is not an integer) and
`TableError(DetsError)` (shared DETS failures). Update call sites that match
on the error:

```gleam
// Before
case set.update_counter(table, "hits", 1) {
  Ok(n) -> ...
  Error(slate.NotFound) -> ...
  Error(_) -> ...
}

// After
case set.update_counter(table, "hits", 1) {
  Ok(n) -> ...
  Error(set.CounterValueNotInteger) -> ...
  Error(set.TableError(slate.NotFound)) -> ...
  Error(set.TableError(_)) -> ...
}
```

**New `DetsError` variant: `TableNamePoolExhausted`** — Returned when all
internal table name slots are in use. Close unused tables to free slots.

**New helpers: `slate.error_code` and `slate.error_message`** — Return a
stable machine-readable code or a concise user-facing description for any
`DetsError`, useful for logging and user-facing output without matching
on every variant.

#### Removed `Kind` type and `TableInfo.kind` field.

`Kind` was an enum (`Set | Bag | DuplicateBag`) returned in `TableInfo`.
Since callers already know the table type from the module they imported, it
was unnecessary. Remove any match on `table_info.kind` or pattern
`TableInfo(kind:, ..)`. The `file_size` and `object_count` fields are
unchanged.

#### Renamed `bag.insert` to `bag.insert_new`; added idempotent `bag.insert`.

Previously `bag.insert` rejected duplicate key-value pairs (returning
`Error(KeyAlreadyPresent)`). It has been replaced by two functions that make
the behavior explicit:

- `bag.insert` — now idempotent; silently ignores duplicate key-value pairs
- `bag.insert_new` — returns `Error(KeyAlreadyPresent)` if the exact pair exists

If you relied on `bag.insert` to detect duplicates, switch to
`bag.insert_new`. If you didn't care about duplicates, no change needed —
`bag.insert` still works but now silently deduplicates instead of erroring.

#### Added labels to `open_with` and `open_with_access` across all table modules.

Call sites using positional arguments must now use labels:

```gleam
// Before
set.open_with("data/cache.dets", ForceRepair,
  decode.string, decode.int)

// After
set.open_with(path: "data/cache.dets", repair: ForceRepair,
  key_decoder: decode.string, value_decoder: decode.int)
```

`open` and `with_table` are unchanged — only `open_with` and
`open_with_access` gained labels on the `path`, `repair`, and `access`
parameters.

#### DETS file-open errors and fold callback failures now surface explicitly.

`slate.DetsError` now exposes `NotADetsFile` and `NeedsRepair` for those
known DETS open failures instead of routing them through
`UnexpectedError(_)`. `fold` and `fold_results` also now re-raise callback
exceptions instead of wrapping them as DETS errors.

Before:

```gleam
case set.open_with(
  path: path,
  repair: slate.NoRepair,
  key_decoder: key_decoder,
  value_decoder: value_decoder,
) {
  Error(slate.UnexpectedError(_)) -> handle_open_failure()
  other -> other
}

case set.fold(table, initial, fun) {
  Error(slate.UnexpectedError(_)) -> handle_callback_failure()
  other -> other
}
```

After:

```gleam
case set.open_with(
  path: path,
  repair: slate.NoRepair,
  key_decoder: key_decoder,
  value_decoder: value_decoder,
) {
  Error(slate.NeedsRepair) -> repair_or_retry()
  Error(slate.NotADetsFile) -> handle_invalid_file()
  other -> other
}

// Callback exceptions now propagate instead of being returned as DetsError.
let fold_result = set.fold(table, initial, fun)
let fold_results_result = set.fold_results(table, initial, fun)
```

Callback-raised failures from `fold` and `fold_results` are no longer
returned as `DetsError`. If you previously matched
`Error(slate.UnexpectedError(_))` for callback failures, move that handling
outside the fold callback and ensure the callback itself does not raise.


### Added

#### Add `fold_results` to `set`, `bag`, and `duplicate_bag` modules.

Unlike `fold`, decode failures do not abort the traversal. Each entry is
presented to the callback as `Ok(#(key, value))` on success or
`Error(decode_errors)` on failure, letting the caller skip, collect, or
partition bad records instead of losing the entire read. DETS-level errors
(e.g., the table is not open) still fail the entire operation via the outer
`Result`.

```gleam
// Skip entries that fail to decode, keeping only valid ones:
set.fold_results(table, [], fn(acc, entry) {
  case entry {
    Ok(#(k, v)) -> [#(k, v), ..acc]
    Error(_) -> acc
  }
})
```

#### Added `dotes`, an example note-taking CLI in `examples/dotes/`.

Demonstrates using all three slate table types together: a set for note
storage with `update_counter` for atomic ID generation, a bag for per-note
tags, and a duplicate bag for revision history. Shows how to open and close
multiple tables safely and use `slate.error_message` for user-facing errors.

#### Added `NotADetsFile` and `NeedsRepair` variants to `DetsError`.

`NotADetsFile` is returned when opening a file that exists but is not a valid
DETS file. `NeedsRepair` is returned when a file was not closed cleanly and
`NoRepair` was requested. Both were previously reported as
`UnexpectedError(...)` with an opaque string detail.


### Fixed

#### Path normalization now collapses `.` and `..` segments.

Paths like `./test.dets` and `../cwd/test.dets` that refer to the same file
now correctly resolve to a single table, avoiding spurious `AlreadyOpen`
errors and unnecessary table-name pool consumption. Symlinks are not resolved.

#### Improved error translation for known DETS error tuples.

`{no_such_table, _}` now maps to `TableDoesNotExist` instead of
`UnexpectedError`. `{not_a_dets_file, _}` and `{needs_repair, _}` now map to
the new dedicated `NotADetsFile` and `NeedsRepair` variants respectively.
